Instructions & Comments

  • Load with file protocol (a decent browser will not follow an href with to a file:// URL, so you will have to copy & paste the url to your address bar) 
    file:///Users/chbrown/github/sandbox/html/local-video.html

    This should let you see both static server and file:// resources below.

  • Load with static server 
    http://ghkl/sandbox/html/local-video.html

    The file:// resource below should be broken (restricted). Firefox's error message is off-base, unfortunately, but Chrome puts it nicely: "Not allowed to load local resource: file:///Users/chbrown/github/sandbox/static/sample-ikea.webm"

Nota bene: If you try this in Firefox with a video/mp4 video file, it will not show anything at all and may lock up.

This is because Firefox cannot decode the mp4 format; normally, if presented with a static resource for the movie, it will give a sane error message to this effect, but compounded with the file:// protocol test, it is simply confounded and doesn't know how to handle the combination.

This is not an issue with WebKit, i.e., Chrome and Safari.

Local video source (file: protocol)

HTML:

    <video width="640" height="360" controls>
        <source src="file:///Users/chbrown/github/sandbox/static/sample-ikea.webm" type="video/webm">
    </video>

Static video source (http: protocol)

HTML:

    <video width="640" height="360" controls>
        <source src="../static/sample-ikea.webm" type="video/webm">
    </video>
